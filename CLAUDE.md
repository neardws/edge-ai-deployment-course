# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Docusaurus 3 site publishing a Simplified Chinese course book ("端侧模型量化部署技术专题" — edge AI model quantization and deployment) plus a reveal.js slide deck. The deliverable is the static site, deployed to Cloudflare Pages from `main` (build command `npm run build`, output directory `build`). All course content — chapter text, slide text, navbar/footer labels — is written in Simplified Chinese.

## Commands

- `npm install` — install dependencies (Node >= 20 required)
- `npm run start` — dev server with hot reload
- `npm run build` — production build into `build/`. This is the main validation step: `onBrokenLinks: 'throw'` in `docusaurus.config.ts` makes the build fail on any broken internal link or missing sidebar doc ID.
- `npm run typecheck` — TypeScript check (`tsc`)
- `npm run serve` — serve the built site locally

There is no test suite or linter.

## Architecture

### Course book (`docs/` + `sidebars.ts`)

- Chapters are Markdown files in `docs/`; frontmatter is just `title:`.
- Navigation is hand-maintained in `sidebars.ts` (seven "Part" categories). There is no autogenerated sidebar, so adding a doc requires adding its ID there too.
- Docusaurus strips numeric filename prefixes when computing doc IDs: `docs/02-ptq-qat.md` has ID `ptq-qat`, and the stripped ID is what `sidebars.ts` entries and `/docs/<id>` links use. Self-study/lab chapters (`lab-*.md`, `prerequisites.md`, etc.) have no numeric prefix.
- Internal links are absolute `/docs/<id>` paths. Mermaid diagrams are enabled in Markdown.
- Math: `$...$` / `$$...$$` render via remark-math + rehype-katex (docs plugin only). The KaTeX stylesheet and woff2 fonts are vendored in `static/vendor/katex/` (re-copy from `node_modules/katex/dist/` when upgrading katex). Literal `$` in prose must go in inline code.
- Chapters follow a fixed authoring structure: 建议学时 table, 学习目标, core concepts with code/command examples, hands-on steps, 验收标准, references. The canonical writing standard is described in `docs/self-study-granularity.md`; new or expanded chapters should match it.

### Slides

Two parts: `src/pages/slides.tsx` is only a launcher page. The actual deck is `static/slides/deck/index.html` — a single self-contained reveal.js presentation with slides authored inline as HTML `<section>` elements and styles in a `<style>` block. It loads reveal.js from the vendored copies in `static/vendor/reveal/` (not from the npm package), so it runs as a standalone static page at `/slides/deck/index.html`.

### Labs (`labs/`)

Shell/Python scripts and Markdown result templates that the lab chapters reference. They are not part of the site build; they run on student lab machines (Ubuntu Server + NVIDIA GPU, Jetson). Examples: `labs/scripts/env_check.sh`, `labs/scripts/openai_compatible_smoke_test.py`, `labs/finetuning/train_lora_smoke.py` (intentionally minimal smoke tests, not production recipes).

## Repository rules

- The original Word course outline is intentionally not committed (`*.docx` and `*课程大纲*` are gitignored). Only public content (Markdown, source code, static assets) goes in the repo.
- Never commit lab artifacts: model weights (`*.gguf`, `*.safetensors`, `*.onnx`, `*.pt`, ...), `models/`, `logs/`, `results/`, `llama.cpp/` are all gitignored.
